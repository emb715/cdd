# CDD Quick Reference

> TL;DR - Everything you need to use Context-Driven Development effectively

## üöÄ Essential Commands

| Command | What It Does | When To Use |
|---------|--------------|-------------|
| `/cdd:create-work [desc]` | Create new work item | Starting any new feature, bug fix, or work |
| `/cdd:plan-work [id]` | Generate implementation tasks | After creating work, before coding |
| `/cdd:save-session [id]` | Save session progress | After each work session (end of day) |
| `/cdd:complete-work [id]` | Mark work done, generate summary | When all requirements are met |
| `/cdd:list-work [filters]` | View all work items | Planning, reviewing, finding work |

## üìÑ Document Purposes

| File | What It Is | Update Frequency |
|------|------------|------------------|
| **DECISIONS.md** | Requirements, decisions, context | Create once, update if requirements change |
| **IMPLEMENTATION_PLAN.md** | Task breakdown, roadmap | Generated by `/cdd:plan-work`, rarely manual edits |
| **SESSION_NOTES.md** | Running log of all sessions | After every session via `/cdd:save-session` |
| **IMPLEMENTATION_SUMMARY.md** | Final retrospective | Generated once by `/cdd:complete-work` |

## üéØ Status Values

- `blocked` - Stuck on dependency or issue

## üí° Command Tips

### `/cdd:create-work`
```bash
# Be specific
/cdd:create-work user authentication with OAuth and email

# Not vague
/cdd:create-work auth stuff
```

**Pro Tips:**
- Answer questions thoughtfully - better input = better docs
- Review generated DECISIONS.md before planning
- List any context files you know about

### `/cdd:plan-work`
```bash
# By work ID
/cdd:plan-work 0001

# By fuzzy name
/cdd:plan-work user auth

# By full path
/cdd:plan-work cdd/0001-user-authentication
```

**Pro Tips:**
- Wait for "Go" confirmation before sub-tasks generate
- Review parent tasks first, adjust if needed
- Check that context files are correct
- After generating sub-tasks, update DECISIONS frontmatter (`planned_tasks_total`, `planned_parent_tasks_total`)

### `/cdd:save-session`
```bash
# Explicit work ID
/cdd:save-session 0001

# Auto-detect from conversation
/cdd:save-session
```

**Pro Tips:**
- Save sessions regularly (end of each work session)
- Document decisions with rationale
- Flag blockers as they occur
- Capture session duration + context reacquisition minutes when prompted
- Update DECISIONS frontmatter aggregates (`total_sessions`, `total_hours`, `total_reacquisition_minutes`, `completed_tasks_total`)
- Don't wait until work is complete

### `/cdd:complete-work`
```bash
/cdd:complete-work 0001
```

**Pro Tips:**
- Verify all FR-X requirements met first
- Review SESSION_NOTES.md tells complete story
- Create follow-up work items for deferred features
- Celebrate completion! üéâ

### `/cdd:list-work`
```bash
# All work items
/cdd:list-work

# Filter by status
/cdd:list-work --status=in-progress

# Filter by type
/cdd:list-work --type=bug

# Filter by priority
/cdd:list-work --priority=critical

# Combine filters
/cdd:list-work --status=draft --priority=high

# Dashboard view
/cdd:list-work --dashboard
```

**Pro Tips:**
- Use `/cdd:list-work` each morning to see what's active
- Filter by `--status=draft` to find work needing planning
- Use `--priority=critical` to find urgent items

## üö® Common Mistakes

| Mistake | Fix |
|---------|-----|
| Forgetting to save sessions | Set reminder: save sessions daily |
| Vague work descriptions | Be specific: "User auth with OAuth" not "auth" |
| Not reviewing generated docs | Always review DECISIONS.md before `/cdd:plan-work` |
| Skipping planning | Always `/cdd:plan-work` before coding |
| Marking incomplete work done | Verify all FR-X before `/cdd:complete-work` |
| Not listing context files | Tell AI which files are important |

## ‚ùì Quick FAQ

**Q: Minimum time investment?**
A: 15min create + 5min plan + 2min/session notes = ~25min overhead. Saves 30min+/session.

**Q: Can I skip IMPLEMENTATION_PLAN.md?**
A: Not recommended. Tasks keep you focused and prevent scope creep.

**Q: How detailed should SESSION_NOTES.md be?**
A: Enough to understand what was done and why. 5-10 minutes of notes = hours saved later.

**Q: What if I worked on multiple items in one session?**
A: Save session for each item separately, or save primary item and mention others.

**Q: Can I modify templates?**
A: Yes! Edit templates in `cdd/.meta/templates/` to fit your needs.
